# SECOM 반도체 수율 분석 — 누수 없이 불량 원인 추적하기

> 원인을 추적하면 수율이 따라온다.

반도체 공정 센서 데이터(UCI SECOM)에서 불량을 예측하는 프로젝트입니다. 목표는 높은 점수가 아니라, **데이터 누수 없이 정직하게 불량의 원인을 좁히는 것**입니다.

---

## 1. 문제 정의

UCI SECOM은 실제 반도체 공정의 센서 측정값입니다. 세 가지가 분석을 어렵게 만듭니다.

- **익명화**: 590개 변수가 모두 익명이라 도메인 지식을 직접 쓸 수 없습니다.
- **극심한 불균형**: 전체 1,567개 중 불량은 약 6.6%뿐입니다.
- **다량 결측**: 결측값이 4만 개를 넘습니다.

이 구조에서는 정확도가 무의미합니다. 전부 정상이라고만 예측해도 정확도가 93%에 이르기 때문입니다.

---

## 2. 첫 시도와 첫 벽

기본 설정으로 RandomForest를 돌리고 정확도를 봤습니다. 높게 나왔지만, confusion matrix를 확인하니 불량을 하나도 잡지 못하고 있었습니다.

```
기본 임계값 0.5
confusion matrix:
[[293   0]
 [ 21   0]]
실제 불량 21개 중 0개 탐지 | 재현율 0.000 | F1 0.000
```

원인은 불균형 자체가 아니라 **분류 임계값**이었습니다. 기본값 0.5는 불량 6.6% 데이터에 너무 높은 기준이라, 불량의 예측 확률이 0.5를 넘지 못해 전부 정상으로 분류된 것입니다.

train에서 F1이 최대가 되는 임계값을 찾으니 **0.13**이었습니다. 이를 적용하자 재현율이 0에서 0.52로 바뀌었습니다. 임계값은 고정값이 아니라 데이터로 정하는 변수였습니다.

---

## 3. 전처리 — 근거로 결정하기

### 결측 처리: 통계가 아니라 공정 관점으로

반도체 공정에서 센서 결측은 무작위가 아닌 경우가 많습니다. 설비 점검, 특정 레시피에서의 미측정, 센서 고장으로 한 구간이 통째로 비기도 합니다. 그래서 제거 전에 결측이 신호인지 확인했습니다.

```
로트별 결측률 평균 0.046 | 최대 0.258
결측률 30% 초과 로트: 0개 / 1253개
결측 여부가 불량과 연관 의심되는 열: 0개
```

결측이 특정 로트에 뭉쳐 있지도, 결측 여부가 불량과 연관되지도 않았습니다. 무작위에 가까웠습니다. 따라서 결측을 플래그로 보존할 필요 없이 안전하게 제거·대치했습니다. 만약 패턴이 있었다면 결측 여부를 별도 신호로 살렸을 것입니다.

### 대치 방법과 열 제거

대치는 분포의 치우침(왜도)을 근거로 median을 택했습니다. 0으로 채우는 방식은 원래 0값이 없는 센서에 가짜 신호를 만들기 때문에 배제했습니다.

열은 세 단계로 제거했습니다.

| 단계 | 기준 | 제거 수 |
|---|---|---|
| 상수 / 전부 NaN | 분산 0 또는 전체 결측 | 116 |
| 고결측 | 결측률 50% 초과 | 24 |
| 중복 | 상관 0.9 초과 | 200 |

590개에서 250개로 줄었습니다.

---

## 4. 데이터 누수 차단 (핵심)

많은 공개 분석이 전처리와 피처 선택을 **데이터 분할 전에** 수행합니다. 이 경우 test 정보가 학습에 새어 점수가 부풀려집니다.

이 프로젝트는 **test를 맨 처음 격리**하고, 결측 대치값·스케일링 평균·피처 선택·임계값까지 모든 결정을 train만 보고 내렸습니다. 처리 과정을 파이프라인으로 묶어 교차검증 매 단계에서 누수가 구조적으로 차단되게 했습니다.

```python
pipe = make_pipeline(
    SimpleImputer(strategy='median'),
    StandardScaler(),
    RandomForestClassifier(...)
)
```

test는 마지막에 단 한 번만 사용했습니다.

---

## 5. 검증 — 유행 기법을 그대로 따르지 않기

불균형 데이터에는 SMOTE를 쓰라는 자료가 많아 시도했습니다. 재검증 결과는 다음과 같습니다.

```
기본   | F1 0.254 | 재현율 0.639
SMOTE  | F1 0.290 | 재현율 0.446
```

SMOTE는 F1을 약간 올리지만 재현율을 0.64에서 0.45로 떨어뜨렸습니다. 불량을 놓치지 않는 것이 목적이라 재현율이 중요한데, SMOTE는 그것을 깎았습니다. 그래서 무조건 적용하는 대신, 목적에 따라 선택하는 트레이드오프로 정리했습니다.

---

## 6. 결과 — 정직하게 두 숫자로

최종 격리 test 성능입니다.

| 지표 | 값 |
|---|---|
| MCC | 0.373 |
| 재현율 | 0.524 |
| 정밀도 | 0.344 |
| 특이도 | 0.928 |
| G-mean | 0.697 |

다만 test 불량이 21개로 적어 단일 수치는 변동이 큽니다. 더 안정적인 추정치는 **교차검증 평균(F1 0.295, 재현율 0.434)**입니다. 안정성은 교차검증이, 정직성은 격리 test가 담당하는 구조입니다.

SECOM은 익명·소량·불균형이라 성능 천장이 낮은 데이터입니다. 검토한 일부 공개 분석에서는 누수가 의심되는데도 성능이 크게 높지 않았습니다. 누수를 차단한 상태에서 MCC 0.37에 도달한 것이 이 프로젝트의 의미입니다.

> 지표 선택 근거: 불균형 데이터에서 정확도·F1은 부풀려질 수 있습니다(Chicco & Jurman, 2020). confusion matrix 네 칸을 모두 반영하는 MCC와, 두 클래스를 함께 보는 G-mean을 함께 사용했습니다.

---

## 7. 산업 관점 — 왜 재현율인가

반도체 후공정은 웨이퍼 단계를 모두 거친 부가가치의 끝단입니다. 여기서 불량이 정상으로 유출되면(FN) 앞 공정 비용까지 손실됩니다. 즉 FN 비용이 FP 비용보다 비대칭적으로 큽니다.

이 비대칭을 임계값으로 다뤘습니다.

| 임계값 | 재현율 | 정밀도 | F1 | 점검 대상 |
|---|---|---|---|---|
| 0.10 | 0.619 | 0.265 | 0.371 | 49 |
| 0.15 | 0.476 | 0.385 | 0.426 | 26 |
| 0.20 | 0.143 | 0.300 | 0.194 | 10 |

임계값이 0.20을 넘으면 재현율이 0.14로 급락합니다. 불량을 놓치지 않으려면 0.10~0.15 구간을 유지해야 합니다. 운영값은 실제 FN·FP 비용을 현업과 함께 확정하는 것이 맞습니다.

---

## 8. 업무 적용 — 위험 로트 자동 선별

분석에서 끝내지 않았습니다. 격리한 test를 새로 들어온 로트로 가정해, 모델이 위험 로트를 우선순위로 자동 선별하게 만들었습니다.

```
[자동 스크리닝] 전체 314개 중 32개 점검 권장 (임계값 0.13)
→ 엔지니어는 전체가 아닌 상위 로트에 우선 집중
```

현장에서는 이 함수에 신규 로트를 넣기만 하면 됩니다. SECOM은 정적 데이터라 실시간 입력은 없지만, 구조는 실전에 그대로 적용됩니다.

---

## 9. 원인 추적 — 어떤 센서가 불량을 유발하는가

위험 로트를 선별하는 데 그치지 않고, "왜 위험한가"를 센서 단위로 추적했습니다. SHAP으로 각 센서가 불량 확률에 기여한 정도와 방향을 분해했습니다.

해석용 모델은 누수를 피하기 위해 train만으로 학습하고, SHAP 값은 test에서 계산했습니다. 또 상위 50개로 미리 거른 편향을 피하려고 전처리만 끝난 전체 변수(약 250개)를 사용했습니다.

### 글로벌 — 전체에서 불량을 유발하는 센서

| 센서 | 기여도(평균\|SHAP\|) | 방향 |
|---|---|---|
| 59 | 0.0151 | 값↑ → 불량↑ |
| 103 | 0.0138 | 값↑ → 불량↑ |
| 33 | 0.0102 | 값↑ → 불량↑ |
| 213 | 0.0086 | 값↑ → 불량↑ |

센서 59와 103의 기여도가 두드러졌고, 둘 다 값이 높을수록 불량이 증가하는 방향이었습니다. 단순 중요도와 달리 방향까지 나오므로, 해당 변수를 낮추는 방향의 공정 관리로 바로 연결됩니다.

### 로컬 — 개별 위험 로트의 원인

```
로트 242 (불량확률 0.245): 59(+0.028), 103(+0.017), 64(+0.017)
로트 215 (불량확률 0.245): 59(+0.043), 103(+0.019), 247(+0.008)
로트 44  (불량확률 0.225): 64(+0.025), 103(+0.015), 59(+0.013)
```

개별 위험 로트를 분해해도 센서 59·103·64가 반복해서 주요 원인으로 나왔습니다. **전체 경향(글로벌)과 개별 사례(로컬)가 일치**해, 이 센서들이 불량의 실제 원인일 가능성이 높습니다.

익명 데이터라 센서의 물리적 정체는 알 수 없습니다. 현업에서는 이 센서 ID를 공정 파라미터 태그와 매핑해 어떤 장비·스텝인지 역추적하는 단계로 이어집니다. 이 분석은 "어떤 변수가 불량과 연관되는가"까지 좁힌 것이고, "그 변수가 어떤 공정인가"는 현업 데이터와 연결하는 단계입니다.

### 선행 연구와의 일치

본 분석에서 도출한 센서 59·103·64는 SECOM에 SHAP을 적용한 선행 연구(DATA ANALYTICS 2022, IARIA)에서 핵심 원인 변수로 지목한 F59·F64·F103과 일치합니다. 독립적으로 수행한 분석이 선행 사례와 같은 변수를 짚었다는 점에서, 결과의 신뢰도를 외부에서 확인할 수 있었습니다. 또한 SHAP의 전역·지역 설명을 병행하는 접근은 산업 공정 결함 진단에서 표준적으로 사용되는 방법입니다.

다만 선행 연구는 원인 분석이 목적이라 전체 데이터로 학습해 학습 f1이 1.0이었습니다(누수가 포함된 이상적 조건). 본 분석은 누수를 차단하고 train만으로 학습한 상태에서도 동일한 변수를 도출했습니다. 더 엄격한 조건에서 같은 결론에 도달한 셈입니다.

### 조치 제안 — 임계값 기반 모니터링

원인 센서를 찾는 데 그치지 않고, "어느 값을 넘으면 위험한가"까지 데이터로 확인했습니다. 각 센서에서 SHAP이 양의 기여로 전환되는 지점을 위험 시작 임계값으로 보고, 그 위아래의 실제 불량률을 비교했습니다.

| 센서 | 위험 임계값 | 임계값 이상 불량률 | 미만 불량률 | 분리력 |
|---|---|---|---|---|
| 59 | 5.663 | 23.6% (n=55) | 3.1% (n=259) | 약 8배 |
| 64 | 25.570 | 12.5% (n=40) | 5.8% (n=274) | 약 2배 |
| 103 | -0.008 | 8.9% (n=79) | 6.0% (n=235) | 약함 |

센서 59는 임계값 5.663을 넘으면 불량률이 3%에서 24%로 약 8배 뛰었습니다. 이 값을 모니터링 알람 기준으로 설정하면 해당 공정 변수를 선제적으로 관리할 수 있습니다. 센서 64도 보조 기준으로 유효합니다.

반면 센서 103은 SHAP 기여도는 높지만 단순 임계값으로는 분리가 약했습니다. 단일 값보다 다른 변수와의 상호작용이나 비선형 관계일 가능성이 있어, 이런 변수는 단순 알람이 아닌 복합 조건이 필요합니다. 모든 센서를 같은 방식으로 다루지 않고, 단순 임계값이 통하는 변수와 그렇지 않은 변수를 구분했습니다.

이 임계값은 선행 연구의 값(F59 약 10)이 아니라 본 모델·전처리 기준에서 직접 도출한 값입니다. 임계값 위에서 불량이 급증하는 패턴 자체는 선행 연구와 일치합니다. 센서의 물리적 정체가 확인되면 알람 기준을 넘어 PM·파라미터 조정 같은 구체적 조치로 이어집니다.

## 10. 한계

- test 불량이 21개로 적어 단일 수치의 변동이 큽니다. 교차검증 평균을 주 추정치로 봅니다.
- 익명화 데이터라 변수의 물리적 정체를 알 수 없습니다. 중요 변수를 실제 공정 파라미터와 매핑하는 단계는 현업 데이터가 있어야 가능합니다.
- SECOM 자체의 성능 천장이 낮아, 절대 수치보다 누수 차단과 정직한 평가에 초점을 두었습니다.
